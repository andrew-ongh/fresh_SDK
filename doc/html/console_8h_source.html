40 /*
41  * Serial console provides input and output through one of the UART's.
42  * It uses UART adapter for reading and writing.
43  * Unlike UART adapter functions, writes to serial port can be started in interrupt context, so
44  * console can be used to allow prints from interrupts.
45  * To allow writes from tasks and interrupts, console uses it's own task that does actual
46  * UART hardware access through UART adapter.
47  * Console does not use additional RAM for printing. RAM that is used is allocated during
48  * initialization only.
49  *
50  * Passing data to print is done using circular buffer that is filled by console_write calls.
51  * If data flow is too fast for UART, calls from task will wait, while calls from interrupt can
52  * lose some data.
53  *
54  * If CONFIG_RETARGET is enabled, it will be automatically handled by console instead of standard
55  * retarget implementation.
56  */
