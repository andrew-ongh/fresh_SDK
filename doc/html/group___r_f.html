void hw_rf_request_off ( bool  mode_ble)

Requests that the RF is turned off.

Requests that the RF is turned off, if not already off. The RF will be turned off only if there are no more requests (ie. all requesters have called hw_rf_request_off()) Interrupts must be disabled by the caller.

Parameters
[in]mode_bleTrue, if the rf was needed for ble

void hw_rf_request_recommended_settings ( void  )

Sets parameters according to their recommended values, taking RF state into account.

Acts like hw_rf_set_recommended_settings but makes sure that the RF power domain is on and unconfigured. Interrupts must be disabled by the caller.

bool hw_rf_start_calibration ( void  )

Start Calibration procedure and return.

This will block for some time in order to perform the first part of calibration (IFF, DC offset and the start of gain calib). Interrupts must be disabled by the caller.

Returns
True if calibration was successful, false if not (i.e. iff calib hang)
void hw_rf_start_continuous_wave ( uint8_t  mode,
uint8_t  ch 
)

Start transmitting a continuous wave (unmodulated transmission)

Parameters
[in]modeis the mode to use. 1: BLE, 2 or 3: FTDF (0: Normal, use hw_rf_stop_*)
[in]chis the Channel to transmit on, calculated as: (mode is BLE) ch = (F – 2402) / 2, where F ranges from 2402 MHz to 2480 MHz. Range: 0x00 – 0x27. (mode is FTDF) ch = (F – 2405) / 5, where F ranges from 2405 MHz to 2480 MHz. Range: 0x00 – 0xf.
