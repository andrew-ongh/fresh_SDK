375 /*
376  * \brief Put the system to idle or sleep or blocks in a WFI() waiting for the next tick, if neither
377  * idle nor sleep is possible.
378  *
379  * \detail Puts the system to idle or sleep, if possible. If an exit-from-idle or a wake-up is
380  * needed, it programs the Timer1 to generate an interrupt after the specified idle or sleep
381  * period. Else, the system stays forever in idle or sleep mode.
382  * If neither idle nor sleep is possible, it blocks in a WFI() call waiting for the next
383  * (already programmed) OS tick to hit.
384  *
385  * \param[in] xLowPowerPeriods The number of (prescaled) low power clock periods the OS will be
386  * idle. If it is 0 then the OS indicates that it can block forever waiting for an
387  * external event. If the system goes to sleep, then it can wake up only from an external
388  * event in this case.
389  *
390  * \warning Must be called with interrupts disabled!
391  *
392  */
